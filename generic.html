Redesign & Product Strategy for Axis Bank Website

Project Overview
Client: Axis Bank (India’s top-tier private bank)
Role: Product Owner, Product Lead for AxisBank.com
Product Type: Web Platform
Problem Statement
AxisBank.com faced challenges typical of legacy financial sites:

  • Overwhelming navigation due to an expansive service portfolio
  • Poor mobile responsiveness and performance
  • Inconsistent UI/UX across pages leading to usability friction
  • Low conversion from homepage areas to key services like Open by Axis Bank, loan application, etc.
Goal: Modernize the website to deliver clarity, responsiveness, and intuitive pathways for retail and corporate users alike.
Discovery & Research
Mapped user journeys to highlight pain points—especially in navigation and self-service flows Defined KPIs around bounce rate, page load times, mobile conversion, and digital engagement metrics.
Product Strategy and Execution
Redesigned key flows like Open by Axis Bank onboarding, bill payments, and product discovery Introduced a modular information architecture: clean banners, segmented offerings, improved site hierarchy axisbank.com.siterate.org Axis Bank Coordinated with UI/UX and engineering teams to enforce brand consistency and performance standards Oversaw A/B testing iterations for homepage layouts and conversion funnels
Results
Modernized homepage with slide-banner UI that improved service discoverability by 25% (internally measured) Reduced bounce rates by focusing on simplified navigation and pre-login service access Axis Bank Optimized for mobile, aligning digital experience between app and web platforms Reinforced Axis’s positioning as a leading digital-first banking brand, enhancing customer trust and usability.
Lessons
Enterprise-scale redesigns require strong information architecture governance Consistent UX between mobile app and website builds user familiarity and trust Lean governance and iterative testing—rather than full rewrites—can deliver high impact in digital banking contexts

Reusable UI Component System for Rehau Smarthome App

Project Overview
Client/Product: Rehau Smarthome System — myEedge Android and iOS app
Role: UI Architecture Lead / Frontend Developer
Tool: Stencil.js for building reusable web-components
Product Type: Android and iOS Application
Problem Statement
A consistent and maintainable UI framework was needed to support rapid feature development across the smarthome app. Design inconsistencies were slowing feature rollouts, and tight timelines demanded reusable patterns.
Product Strategy
Designed a UI template system using Stencil.js, enabling encapsulated and reusable components across the app. Coordinated with design and product teams to define guidelines for components like switches, sliders, thermostats, and control layouts. Established documentation and usage patterns for integration into multiple product modules.
Execution
Developed Stencil-based web components delivering consistent UI styles and behavior, boosting development efficiency. Ensured component flexibility for embedding in live smarthome screens while maintaining code modularity. Facilitated onboarding for frontend developers to adopt the component library, reducing duplicate work and visual mismatches.
Results
UI consistency improved across modules, ensuring a cohesive user experience. Reduced development effort for UI screens by up to 40%, accelerating new feature timelines. Simplified future design updates, since changes made in central components propagated across the app seamlessly.
Lessons
A well-documented component system is powerful for scaling consistent UX. Close alignment with product and design teams during component specification saves rework. Choosing lightweight frameworks like Stencil.js promotes flexibility, performance, and maintainability.
